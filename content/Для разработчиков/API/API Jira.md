## 1. Общая информация

Jira предоставляет REST API (версия v3), которое позволяет получать и изменять данные, связанные с задачами, пользователями и проектами.

API работает по HTTP(S) и возвращает данные в формате JSON.

Доступ осуществляется через авторизацию (API token / OAuth 2.0).

**Назначение данных:**
- Получение исходной информации о задачах и активности команды

**Использование в системе:**
- Основной источник данных для аналитики, дашбордов, отчётов и AI-ассистента
## 2. Ключевые endpoints и данные

### 2.1 Задачи (Issues)

**Используемые endpoints:**
- GET /rest/api/3/issue/{issueIdOrKey}  - получение полной информации по задаче
- GET /rest/api/3/search  - получение списка задач с фильтрацией (JQL)
- POST /rest/api/3/issue  - создание задачи
- PUT /rest/api/3/issue/{issueIdOrKey}  - обновление задачи

**Получаемые данные:**
- идентификатор (id, key)
- название (summary)
- описание (description)
- статус (status)
- тип задачи (issueType)
- исполнитель (assignee)
- автор (reporter)
- приоритет (priority)
- даты (created, updated, dueDate)
- метки (labels)
- иерархия (parent, subtasks)
- связи (issuelinks)
- вложения (attachments)

**Назначение данных:**
- Описание задач и их текущего состояния

**Использование в системе:**
- получение данных для дашбордов
- анализ загрузки сотрудников
- создание задач через AI-бота
- обновление задач 

###  2.2 Активность по задачам

**Используемые endpoints:**
- GET /rest/api/3/issue/{issueIdOrKey}/changelog  - история изменений
- GET /rest/api/3/issue/{issueIdOrKey}/worklog  - учёт времени

**Получаемые данные:**
- история изменений (переходы по статусам, изменения полей)
- журнал работ (учет времени, затраченного на задачу)

**Назначение данных:**
- Отслеживание процесса работы над задачей

**Использование в системе:**
- анализ прогресса
- расчёт времени выполнения
- построение отчетов
- анализ коммуникации через AI 

### 2.3 Управление задачами

**Используемые endpoints:**
- PUT /rest/api/3/issue/{issueIdOrKey}/assignee  - назначение исполнителя
- GET /rest/api/3/issue/{issueIdOrKey}/transitions  - доступные статусы
- POST /rest/api/3/issue/{issueIdOrKey}/transitions  - смена статуса

**Назначение данных:**
- Изменение состояния задач

**Использование в системе:**
- управление задачами через интерфейс
- автоматизация через AI-бота

### 2.4 Метаданные

**Используемые endpoints:**
- GET /rest/api/3/issue/createmeta  - данные для создания задач
- GET /rest/api/3/field  - список полей (включая custom fields, Story Points  )
- GET /rest/api/3/project  - список проектов

**Назначение данных:**
- Получение структуры и конфигурации Jira

**Использование в системе:**
- маппинг проектов
- определение Story Points
- настройка интеграции и аналитики
