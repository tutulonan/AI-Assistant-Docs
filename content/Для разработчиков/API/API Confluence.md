## 1. Общая информация

Confluence предоставляет REST API, позволяющее получать и изменять данные, связанные с контентом (страницы, пространства, пользователи и др.). API работает по HTTP(S) и возвращает данные в формате JSON. Доступ осуществляется через авторизацию (Basic auth / API token / OAuth).

Назначение данных:  
Получение информации о документации, знаниях и структуре проектов.

Использование в системе:

Источник данных для AI-ассистента, поиска знаний, анализа документации и протоколов.

## 2. Ключевые endpoints и данные

### 2.1 Страницы (Pages)

Используемые endpoints:
- GET /rest/api/content/{id} - получение страницы
- GET /rest/api/content - получение списка страниц
- POST /rest/api/content - создание страницы
- PUT /rest/api/content/{id} - обновление страницы

Получаемые данные:
- идентификатор (id)
- заголовок (title)
- содержимое (body, storage/HTML)
- статус (current/draft)
- автор (creator)
- дата создания и обновления
- родительская страница (parent)
- дочерние страницы (children)
- пространство (space)
- версия страницы (version)

Назначение данных:  
Хранение и получение документации.

Использование в системе:
- AI-поиск по знаниям
- анализ документации
- генерация ответов AI
- создание и обновление страниц

### 2.2 Пространства (Spaces)

Используемые endpoints:
- GET /rest/api/space - список пространств
- GET /rest/api/space/{spaceKey} - информация о пространстве
- POST /rest/api/space - создание пространства
- PUT /rest/api/space/{spaceKey} - обновление пространства

Получаемые данные:
- ключ пространства (key)
- название (name)
- описание (description)
- тип (global/personal)
- статус (current/archived)
- метаданные (labels и др.)

Назначение данных:  
Структурирование документации по проектам.

Использование в системе:
- маппинг проектов
- группировка данных
- навигация по знаниям

### 2.3 Поиск (Search / CQL)

Используемые endpoints:
- GET /rest/api/content/search?cql=...

Получаемые данные:
- страницы по фильтрам (по тексту, пространству, типу)
- результаты поиска с метаданными

Назначение данных:  
Поиск информации в Confluence.

Использование в системе:
- AI-ответы
- поиск по документации
- выборка данных для анализа

### 2.4 Дополнительные данные (Relations и активность)

**Используемые endpoints:**
- GET /rest/api/content/{id}?expand=history
- GET /rest/api/content/{id}?expand=version
- GET /rest/api/content/{id}?expand=children

**Получаемые данные:**
- история изменений
- версии страниц
- структура (родители/дети)

**Назначение данных:**  
Отслеживание изменений и структуры документации.

**Использование в системе:**
- анализ изменений
- построение структуры знаний
- AI-контекст

## 3. Ограничения API

- Доступ к страницам зависит от прав пользователя (возвращаются только доступные страницы)
- Пагинация (limit, start / cursor) обязательна при больших выборках
- Формат контента сложный (HTML/storage), требует обработки
- Нет прямой связи с Jira/Git - нужна интеграция на backend
- Поиск (CQL) ограничен и требует правильной настройки фильтров

## 4. Ключевые используемые endpoints (MVP)

- /content - основной источник страниц
- /content/{id} - детальная информация
- /content/search - поиск (CQL)
- /space - пространства
- expand=history/version - изменения
- expand=children - структура
