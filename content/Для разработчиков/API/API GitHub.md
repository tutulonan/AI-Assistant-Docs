
**API GITHUB:** https://docs.github.com/en/rest

## 1. Общая информация

GitHub API предоставляет доступ к данным системы контроля версий, включая информацию о репозиториях, коммитах, pull request’ах и активности разработчиков. API работает по HTTP(S) и возвращает данные в формате JSON. Доступ осуществляется через авторизацию (Personal Access Token / OAuth 2.0).

**Назначение данных:**  
Получение информации о фактической разработке и активности команды в репозиториях.

**Использование в системе:**

Источник данных для анализа активности разработчиков, сопоставления с задачами Jira и оценки прогресса выполнения работ.

## 2. Ключевые endpoints и данные

### 2.1 Репозитории (Repositories)

**Используемые endpoints:**
- GET /user/repos — список репозиториев пользователя
- GET /repos/{owner}/{repo} — информация о репозитории

**Получаемые данные:**
- название репозитория
- владелец
- язык программирования
- дата создания и обновления
- количество веток и звезд

**Назначение данных:**  
Описание структуры проектов в системе контроля версий.

**Использование в системе:**
- привязка репозиториев к проектам
- анализ технологического стека

### 2.2 Коммиты (Commits)

**Используемые endpoints:**
- GET /repos/{owner}/{repo}/commits — список коммитов
- GET /repos/{owner}/{repo}/commits/{sha} — детали коммита

**Получаемые данные:**
- идентификатор коммита (SHA)
- автор
- дата и время
- сообщение коммита
- количество изменений (additions, deletions)

**Назначение данных:**  
Отслеживание активности разработки и изменений в кодовой базе.

**Использование в системе:**
- анализ активности разработчиков
- сопоставление с задачами Jira
- оценка прогресса разработки 

### 2.3 Pull Requests

**Используемые endpoints:**
- GET /repos/{owner}/{repo}/pulls — список pull request’ов
- GET /repos/{owner}/{repo}/pulls/{pull_number} — детали PR

**Получаемые данные:**
- автор
- статус (open/closed/merged)
- дата создания и закрытия
- количество изменений
- связанные ветки

**Назначение данных:**  
Контроль процесса code review и интеграции изменений.

**Использование в системе:**
- анализ процесса ревью
- оценка времени выполнения задач
- контроль качества разработки


### 2.4 Активность разработки

**Используемые endpoints:**
- GET /repos/{owner}/{repo}/stats/contributors — статистика по разработчикам
- GET /repos/{owner}/{repo}/stats/commit_activity — активность коммитов

**Получаемые данные:**
- количество коммитов по пользователям
- активность по неделям
- распределение изменений

**Назначение данных:**  
Анализ общей активности команды разработки.

**Использование в системе:**
- оценка загрузки разработчиков
- построение аналитики активности
- выявление периодов низкой активности

### 3. Ограничения API

- Ограничение на количество запросов (rate limit)
- Пагинация при больших объемах данных
- Задержка в формировании статистики (для некоторых endpoints)
- Требуется корректная настройка прав доступа к репозиториям

### 4. Ключевые используемые endpoints (MVP)

- /user/repos — список репозиториев
- /repos/{owner}/{repo}/commits — история изменений
- /repos/{owner}/{repo}/pulls — pull requests
- /repos/{owner}/{repo}/stats/* — аналитика активности**
