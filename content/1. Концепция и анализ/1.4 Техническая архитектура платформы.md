
### Серверная часть (Backend)

- Язык: Python
- Фреймворк: FastAPI
- База данных: PostgreSQL
- Назначение:
	-Обработка запросов от веб-клиента и трей-приложения.
	-Взаимодействие с интеграциями Jira, Git и Confluence через API.
	-Хранение данных о проектах, задачах, пользователях, метриках.
	-Обработка и хранение логов и истории действий пользователей.


### Клиентская часть (React + Electron)

**Веб-клиент (React):**
- Основной интерфейс через браузер.
- Дашборд с метриками, просмотр проектов.
- Плавающий AI-чат для быстрого получения информации .
- Связь с сервером через REST API и WebSocket для реального времени.

**Десктоп-клиент (React + Electron):**
- Мини-приложение в трее с быстрым доступом к чат-боту и уведомлениям.
- Автозапуск с системой для мгновенного мониторинга проектов.

**Синхронизация состояния с веб-клиентом:**
- При логине подтягиваются проекты, задачи и уведомления.
- Любое действие (создание задачи через AI, отметка выполненной) обновляет сервер и мгновенно отражается на веб-клиенте.
- Поддержка локальных уведомлений (push) и системного трея для мгновенного оповещения.

**Синхронизация между вебом и трей-клиентом:**
- Через WebSocket или REST API.
- Общий контекст диалога AI, статусы задач и уведомления обновляются мгновенно.
- Локальный кэш в Electron позволяет показывать актуальные данные даже при кратковременной потере соединения.

### AI-компонент

**Назначение:** интеллектуальный чат-бот для анализа данных и автоматизации задач.

**Хостинг:** отдельный микросервис (можно на FastAPI или на облаке для масштабируемости).

**Источники данных:**
- Jira — задачи, исполнители, сроки, статусы.
- Git — коммиты, pull requests, ревью, CI/CD статус.
- Confluence — документация, протоколы встреч, база знаний.

**Функции:**
- Понимание вопросов на русском языке (NLU).
- Классификация намерений (Intent Classification).
- Извлечение сущностей (Entity Extraction).
- Поиск по базе знаний (Knowledge Base Retrieval).
- Создание и обновление задач в Jira.

  
### Потоки данных

- Пользователь взаимодействует с вебом или трей → отправляет запрос на сервер.
- Сервер FastAPI → обрабатывает данные, проверяет права доступа.
- Если действие требует AI → запрос к AI-компоненту.
- AI → получает контекст, ищет данные в Jira/Git/Confluence → возвращает результат серверу.
- Сервер → обновляет базу PostgreSQL и отправляет ответ клиенту (веб/трей).
- Состояние синхронизируется между трей и вебом через WebSocket / REST API.
